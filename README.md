Порівняльний аналіз алгоритмів сортування: Insertion Sort, Merge Sort та Timsort 

В даному завданні виконано порівняльний аналіз трьох алгоритмів сортування:
1. Insertion Sort (сортування вставками)
2. Merge Sort (сортування злиттям)
3. Timsort (гібридний алгоритм, який використовується Python у функціях sorted() та list.sort())

Мета роботи — емпірично перевірити теоретичні оцінки складності алгоритмів та показати переваги Timsort, який поєднує сильні сторони сортування злиттям і сортування вставками.

Сценарій тестування:
Для вимірювання часу виконання використано модуль timeit, тест здійснювався на випадково згенерованих масивах розміром:
1000 елементів
5000 елементів
10000 елементів
На кожен алгоритм — 10 запусків.
Результат — середній час виконання.

Результати тестування
1. 1000 елементів:

| Алгоритм                | Час (сек) |
| ----------------------- | --------- |
| sorted() (Timsort)    | 0.00064   |
| list.sort() (Timsort) | 0.00052   |
| Insertion Sort          | 0.18236   |
| Merge Sort              | 0.01417   |

2. 5000 елементів

| Алгоритм                | Час (сек) |
| ----------------------- | --------- |
| sorted() (Timsort)    | 0.00369   |
| list.sort() (Timsort) | 0.00358   |
| Insertion Sort          | 4.40477   |
| Merge Sort              | 0.08613   |

3. 10000 елементів

| Алгоритм                | Час (сек) |
| ----------------------- | --------- |
| sorted() (Timsort)    | 0.00814   |
| list.sort() (Timsort) | 0.00789   |
| Insertion Sort          | 17.90059  |
| Merge Sort              | 0.18539   |

Аналіз результатів:
1. Insertion Sort показав квадратичну складність O(n^2). 
Час виконання збільшується дуже швидко. Наприклад, при переході від 5000 до 10000 елементів час збільшується в 4 рази, що характерно для O(n^2).
2. Merge Sort показав стабільний ріст O(n log n).
Збільшення часу для 10000 елементів 0.185 с — що в десятки разів швидше ніж insertion sort.
3. Timsort став найшвидшим алгоритмом у всіх тестах.
Працює в 20 разів швидше, ніж merge sort, і в сотні разів швидше за insertion sort. Timsort стабільно демонструє найкращі результати.

Висновок:
Емпіричні дані повністю підтверджують, що алгоритм Insertion Sort має найгіршу ефективність на великих масивах.
Merge Sort працює швидко та стабільно, відповідно до O(n log n).
Timsort показує найкращий результат завдяки адаптивності й комбінації двох алгоритмів.
Саме тому програмісти використовують вбудовані алгоритми Python (sorted(), sort()), а не реалізують сортування вручну.
